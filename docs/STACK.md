# Stack technique — versions, statut & revue sécurité

> **Rôle de ce document.** `SPEC.md §4` justifie _qualitativement_ chaque choix de stack (pourquoi
> Next, pourquoi Recharts, pourquoi `@node-rs/argon2`…). Ce document le **complète** sur l'axe que
> le brief Session 1 exige explicitement — « stack choisie après recherches web sérieuses, **versions
> documentées**, durable et pérenne » : pour chaque librairie clé, la **version installée**, la
> **version stable courante**, un **statut** (current / behind / beta-risk), une **revue CVE/sécurité**,
> et un **verdict** actionnable.
>
> **Méthode.** Versions installées lues dans `apps/web/package.json`. Versions courantes + CVE
> vérifiées par recherche web + MCP context7 le **2026-06-24** (sources citées en bas). Toute
> affirmation de version/CVE ici a été confrontée à une source primaire ; les claims non confirmées
> sont marquées comme telles.

## Synthèse

- **Sécurité** : aucune CVE non corrigée n'affecte la stack à 2026-06-24. Le seul point dur est le
  release Next mai 2026 — **et Fxmily est sur la version qui le corrige** (16.2.6, qui inclut le fix
  SSRF `CVE-2026-44578` de 16.2.5).
- **Un seul risque suivi** : la régression React 19.2.6 « FormData / Server Actions ». Réelle en
  amont, mais (a) non corrigeable proprement au niveau app (Next vendorise `react-server-dom`),
  (b) **non manifeste chez Fxmily** (CI e2e verte sur les formulaires). → suivi owner, pas de bump
  risqué (cf. tableau).
- **Tout le reste** est à jour ou attrapé par caret (`tailwind 4.3.1`, `vitest 4.1.9`).
- **1 dette technique** non urgente : `recharts <Cell>` deprecated → migrer vers `shape` avant v4.

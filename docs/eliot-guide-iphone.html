Smoke test Web Push iPhone — étape par étape

Tu installes Fxmily comme une app sur ton iPhone et tu vérifies que les notifications push arrivent. 10 minutes de tes mains, sur un iPhone réel (pas simulateur).

⏱ ~10 min total 📱 iPhone iOS 18.4+ requis 🌐 Safari obligatoire (pas Chrome iOS) 🇫🇷 France OK (DMA EU PWA Safari fix tardif)
À lire avant de commencer

Web Push iOS exige 3 conditions cumulées : (1) iOS ≥ 18.4, (2) PWA installée via Safari (pas onglet, pas autre browser), (3) permission "Autoriser" acceptée à la 1re demande. Si tu rates l'une, désinstalle et recommence — iOS donne UNE SEULE chance par installation pour la permission.

1

Vérifier la version iOS

⏱ 30 s
Ce que tu vois : écran d'accueil iPhone (icônes app, dock en bas avec 4 apps).

Action

  1. Ouvre Réglages (icône grise, engrenages) — souvent page 1, parfois dans Bibliothèque d'apps (swipe gauche jusqu'au bout).
  2. Touche Général (icône grise avec engrenage à l'intérieur, ~5e ligne, sous "Concentration" et "Temps d'écran").
  3. Touche Informations (1re ligne du bloc Général).
  4. Lis la ligne "Version du logiciel" ou "Version iOS".

Validation

Doit afficher 18.4 ou plus (ex : 18.7.8, 26.1). Si < 18.4 → Réglages → Général → Mise à jour logicielle et installe (compte 30 min + redémarrage).

iPhone non compatible iOS 18

iPhone 8/X et plus anciens (limite hardware A11) ne peuvent pas monter à 18. Modèles compatibles : iPhone XS, XR, 11, 12, 13, 14, 15, 16, SE 2, SE 3.

2

Vérifier Safari (ou s'assurer qu'il existe)

⏱ 15 s

Sur iOS, Safari n'a pas de version indépendante d'iOS — ils partagent WebKit. Si tu es en iOS 18.4+, Safari est en 18.4+.

Action

Confirme que l'icône Safari (boussole bleue/blanche) est présente — sur écran d'accueil ou Bibliothèque d'apps.

Pièges
  • Safari peut être caché par Temps d'écran : Réglages → Temps d'écran → Restrictions de contenu et confidentialité → Apps autorisées → vérifier que Safari est ON. S'il est OFF, l'installation PWA est impossible.
  • En France, Web Push fonctionne uniquement via Safari (pas Chrome iOS, pas Edge iOS).
3

Aller sur app.fxmilyapp.com

⏱ 30 s

Action

  1. Touche l'icône Safari.
  2. Touche la barre d'adresse (centre-bas en iOS 18, ou haut selon réglage perso).
  3. Tape app.fxmilyapp.com (sans https://, Safari l'ajoute auto).
  4. Touche Aller (touche bleue clavier).

Validation

La home Fxmily charge — hero "splash" sombre, logo Fxmily, CTA "Se connecter". Si erreur réseau ou cert : signale-le, ne continue pas.

4

Installer en PWA — "Sur l'écran d'accueil"

⏱ 1 min
Ce que tu vois : la page Fxmily affichée. En bas de l'écran (iOS 18 défaut), une barre de boutons : ‹ (retour), › (avancer), icône partage (carré + flèche montante), icône onglets (deux carrés superposés).

Action

  1. Touche l'icône partage (carré + flèche pointant vers le haut, bas-centre sur iOS 18 avec barre en bas, ou bas-droite si barre en haut).
  2. Une feuille de partage glisse depuis le bas, occupant ~70% de l'écran. En haut : aperçu page (favicon + "Fxmily" + URL). Rangée 1 : contacts AirDrop. Rangée 2 : icônes apps. En dessous : liste verticale d'actions (Copier, Ajouter aux favoris, Liste de lecture, Marque-page, Sur l'écran d'accueil, Annoter…).
  3. Scroll vers le bas dans la feuille jusqu'à voir "Sur l'écran d'accueil" (icône carré avec +).
  4. Touche "Sur l'écran d'accueil".
  5. Vue d'aperçu : icône Fxmily (gauche) + champ texte "Fxmily" pré-rempli (droite). Touche "Ajouter" (haut-droite, texte bleu).

Validation

Retour automatique à l'écran d'accueil. Une nouvelle icône "Fxmily" apparaît (logo Fxmily ou capture du favicon).

Pièges
  • "Sur l'écran d'accueil" n'apparaît pas : scroll tout en bas → "Modifier les actions" → active "Sur l'écran d'accueil" (toggle vert) → réessaye.
  • Tu navigues depuis Messages/Mail : feuille partage limitée. Touche les 3 points "..." → "Ouvrir dans Safari".
5

Renommer l'app installée (optionnel)

⏱ 5 s

L'étape 4 propose un champ texte "Fxmily". Laisse "Fxmily" par défaut. Pas de renommage requis. Si tu veux raccourcir : tape directement, supprime, retape.

L'identité PWA est dans le manifest.json côté serveur, pas dans le label local.

6

Ouvrir l'app DEPUIS l'écran d'accueil (CRITIQUE)

⏱ 10 s
Pourquoi c'est critique

iOS Safari n'expose Web Push qu'en mode standalone. Si tu ouvres app.fxmilyapp.com dans un onglet Safari, window.Notification reste indisponible et l'API pushManager.subscribe() rejette. Seul le lancement via icône Home Screen donne le contexte PWA standalone.

Action

  1. Quitte Safari (swipe up depuis le bas + relâche au milieu, ou bouton Home).
  2. Localise l'icône Fxmily sur l'écran d'accueil (la page où elle a été ajoutée — dernière page utilisée). Si tu ne la trouves pas : swipe droite jusqu'à la Bibliothèque d'apps et tape "Fxmily".
  3. Touche l'icône.

Validation

L'app s'ouvre sans la barre d'adresse Safari ni la barre de boutons en bas. Plein écran. C'est le mode standalone.

Pièges
  • Si tu vois la barre Safari + barre de boutons en bas : tu es en onglet (étape 3 d'install ratée). Désinstalle (touche longue icône → "Supprimer le marque-page" / "Supprimer du Home Screen") et refais étape 4.
  • Si "Ouvrir dans Safari" en haut de l'app : manifest.json serveur a "display": "browser" au lieu de "standalone". Bug serveur, pas toi.
7

Login admin

⏱ 30 s

Action

  1. Touche le champ Email → tape l'email admin.
  2. Touche le champ Password → tape le password.
  3. Touche "Se connecter".

Auto-save iOS Keychain ?

Oui, si :

  • iCloud Keychain activé (Réglages → [Toi] → Mots de passe → Mots de passe et trousseau → toggle "Synchroniser ce iPhone" ON).
  • Auto-remplissage activé (Réglages → Apps → Mots de passe → Auto-remplissage activé).

Au-dessus du clavier, une barre "Enregistrer le mot de passe ?" apparaîtra après login réussi → "Enregistrer" ou "Pas maintenant".

Validation

Redirection vers le dashboard Fxmily (home authentifiée).

8

Aller à /account/notifications

⏱ 20 s

Action

Selon la nav Fxmily V1, soit :

  • Touche l'icône profil (rond, top-right) → menu déroulant → "Notifications".
  • Soit menu hamburger (3 lignes top-left) → "Compte" → "Notifications".
  • Sinon : navigation interne Compte → Notifications.

Sur la page

Un toggle "Activer les notifications push" (état par défaut : OFF). Touche-le.

Validation

Toggle passe en vert → un popup système iOS apparaît (étape 9).

9

Permission Safari iOS

⏱ 15 s
Ce que tu vois : popup système iOS centré, titre « Fxmily souhaite vous envoyer des notifications », sous-titre "Les notifications peuvent être des alertes, des sons et des pastilles. Elles peuvent être configurées dans Réglages." Deux boutons : "Refuser" (gauche) et "Autoriser" (droite, bleu).

Action

Touche "Autoriser".

Validation

Popup disparaît. Le toggle reste vert dans l'app. La page peut afficher "Notifications activées" ou similaire.

Une seule chance

Une seule chance par installation PWA. Si tu touches "Refuser" par erreur, le popup ne reviendra jamais. Pour récupérer :

  • Méthode 1 : Réglages → Notifications → scroll → trouve "Fxmily" → toggle "Autoriser les notifications" ON. Mais sur iOS 18.4+ avec PWA, Fxmily n'apparaît dans cette liste que si permission a été accordée au moins une fois.
  • Méthode 2 (fallback) : touche longue icône Fxmily écran d'accueil → "Supprimer du Home Screen" → "Supprimer". Refais étapes 3-9.
10

Vérifier dans Réglages iOS

⏱ 30 s

Action

  1. Quitte la PWA (swipe up).
  2. Réglages (icône grise engrenages).
  3. Touche Notifications (bloc haut, icône rouge bell).
  4. Scroll vers le bas dans la liste alphabétique des apps. Trouve "Fxmily".
  5. Touche "Fxmily".

Validation

  • "Autoriser les notifications" = ON (toggle vert).
  • Style d'alerte : "Bannières", "Sons", "Pastilles" tous activables.
  • Si Fxmily n'apparaît pas dans la liste = la permission n'a pas été accordée (popup étape 9 raté ou refusé). Retour étape 4 (réinstall).
11

Forcer un push test

⏱ 1 min

Action

Depuis ton ordinateur, lance :

curl -X POST https://app.fxmilyapp.com/api/cron/dispatch-douglas \
  -H "X-Cron-Secret: $CRON_SECRET" \
  -H "Content-Type: application/json"

Remplace $CRON_SECRET par la valeur réelle (depuis ton tokens.local.env). Réponse attendue : JSON { "ok": true, "dispatched": N } (200 OK).

Validation côté iPhone

  • Dans 5 à 30 sec, l'iPhone reçoit une notification push avec une citation Mark Douglas.
  • Si écran allumé : bannière en haut.
  • Si écran verrouillé : notif sur lock screen.
  • Touche la notif → ouvre Fxmily PWA directement à la page concernée (deep-link).
Pièges
  • Réponse 200 mais pas de notif → checkpoints étape 12.
  • Réponse 401/403 → mauvais X-Cron-Secret (vérifier env).
  • Réponse 500 → log Vercel + Sentry.
12

Troubleshooting complet

⏱ cas par cas

A · Permission refusée par accident

  1. Réglages → Notifications → Fxmily → si présent, toggle ON.
  2. Si absent : désinstaller PWA + réinstaller (étapes 4 + 9).

B · PWA pas reconnue comme installée (mode browser au lieu de standalone)

Symptôme : barre Safari visible quand tu ouvres "Fxmily" depuis Home Screen, ou bandeau "Ouvrir dans Safari".

Cause : manifest.json serveur expose "display": "browser" au lieu de "standalone". Bug serveur, pas toi.

Fix : vérifier apps/web/public/manifest.json → "display": "standalone". Redéployer. Désinstalle + réinstalle PWA iPhone.

C · Notifications ne s'affichent pas (push arrive serveur mais pas écran)

Vérifie dans l'ordre :

  1. Mode Avion OFF : Centre de contrôle (swipe haut depuis coin haut-droite iPhone X+) → icône avion PAS orange.
  2. Concentration / Focus mode OFF : Centre de contrôle → tuile "Concentration" (icône lune/voiture/personne) éteinte. iOS 18 a une feature "Personnel" qui s'active toute seule via IA — vérifie : Réglages → Concentration → Personnel → désactive "Filtrage intelligent" et "Silence intelligent".
  3. Mode Économie d'énergie OFF : Réglages → Batterie → toggle "Mode économie d'énergie" OFF. Note 2026 : ce mode désactive les push web/PWA, pas seulement les retarde. Icône batterie en haut-droite PAS jaune.
  4. Background App Refresh ON : Réglages → Général → Actualisation en arrière-plan → ON (Wi-Fi + cellulaire OK).
  5. Restart Safari / iPhone : redémarrage iPhone (boutons volume haut + power → glisser pour éteindre).
  6. Throttling Apple : si tu n'as pas ouvert la PWA depuis 7-14 jours et pas tapé sur les push reçus → Apple bride et désabonne le endpoint silencieusement. Symptôme : subscription.endpoint change ou disparaît côté DB. Fix : ouvre la PWA, le SW renouvelle l'abonnement au prochain pushsubscriptionchange.

D · Apple Declarative Web Push (8030) vs Classic

Safari 18.4 (mars 2025) a introduit Declarative. Côté serveur, on envoie un payload JSON avec clé top-level "web_push": 8030 qui agit comme "magic key" → notification s'affiche sans Service Worker. Fallback : si pas reconnu, le SW prend le relais (classique).

Côté toi : aucune différence visible. Backend Fxmily fait les deux.

E · Mode Focus / Ne pas déranger (iOS 18)

iOS 18 a redessiné Concentration avec IA. Symptôme : "Personnel" s'active automatiquement à des horaires que tu n'as pas programmés. Fix : Réglages → Concentration → chaque profil → vérifier "Activation intelligente" OFF si non voulu, et autoriser Fxmily dans "Apps autorisées" → "+" → Fxmily.

F · Battery Saver et push silent

Confirmé : Mode économie désactive les push (web et natifs) pendant qu'il est actif. Pas un retard, un blocage. Charge l'iPhone à >80% pour désactivation auto, ou désactive manuellement.

G · Service Worker debug iPhone

Impossible d'inspecter directement depuis iPhone. Options :

  1. Safari Web Inspector via Mac USB : connecte iPhone à Mac via câble Lightning/USB-C → Mac : Safari → Réglages → Avancé → "Afficher menu Développement" → menu Développement → "[ton iPhone]" → "Fxmily" → fenêtre Inspecteur (Console, Réseau, Stockage, Service Workers).
  2. Sans Mac : ajouter logging serveur exhaustif côté SW vers endpoint /api/debug/sw-log.
✓

Checklist finale

  • iOS ≥ 18.4
  • PWA installée via Safari (pas autre browser, pas onglet)
  • App ouverte depuis Home Screen (mode standalone confirmé visuellement)
  • Permission "Autoriser" cliquée
  • Fxmily visible dans Réglages → Notifications
  • Pas de Mode Avion / Focus / Économie d'énergie actif
  • curl POST → 200 OK
  • Notif reçue dans 5-30 sec
Si tous les checkpoints passent

Web Push smoke test = PASS. Fxmily V1 est validé end-to-end côté membre. La cohorte peut être invitée.

Si un checkpoint fail

Section 12 + log Sentry + DB endpoint check (SELECT id, endpoint, last_error_code FROM push_subscriptions WHERE user_id = '...';).

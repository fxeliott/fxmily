📧 Resend — emails transactionnels

Sender pour invitations, magic links, weekly digests. Plan free 2026 : 3 000 emails/mois + 100/jour cap, 1 custom domain, retention 30 jours, pas de CB.

⏱ ~10 min 📍 resend.com 💳 Pas de CB 🇪🇺 Région : Pro only
Correction critique vs guide précédent

DKIM Resend = 1 seul TXT record (PAS 3 CNAMEs *.dkim.amazonses.com). Le pattern 3-CNAME c'est SES direct, pas Resend. Confirmé via doc Resend live 2026-05-09. Si tu as posé 3 CNAMEs ailleurs : supprime-les et pose 1 TXT comme expliqué étape 7.

Région : Pro only

Le multi-region (Frankfurt EU) est Pro only ($20/mo). Sur Free, Resend choisit auto (souvent US). Pour RGPD strict France → upgrade Pro avant Add Domain. Sinon, accepte la région auto (acceptable pour 30 membres V1).

7

DNS Records (le coeur du sujet)

https://resend.com/domains/<uuid>

Titre fxmilyapp.com + badge "Not Started" (gris) ou "Pending" (jaune). Bouton "Verify DNS Records" à droite. Tableau DNS en dessous avec colonnes Type | Name | Value | Priority | TTL | Status.

Structure (3 lignes, parfois 4) — confirmé 2026

#TypeNameValue (extrait)Priority
1 MX send feedback-smtp.us-east-1.amazonses.com (ou eu-central-1 si Pro EU) 10
2 TXT send "v=spf1 include:amazonses.com ~all" —
3 TXT resend._domainkey p=MIGfMA0GCSq... (~210 chars, unique à ton domaine) —
Receiving toggle

Pour Fxmily V1, garde Receiving OFF. Tu envoies, tu ne reçois pas. Toggle "Receiving" en haut de la page = laisser désactivé. Sinon, 4e ligne MX inbound à poser.

Comment copier les valeurs

[BOUTON copy] à droite de chaque cellule "Value", icône carrée (deux rectangles superposés). Clique → toast "Copied!".

DEUX pièges critiques

1. Le champ Name = `send` ou `resend._domainkey` SANS le domaine. Ne colle pas send.fxmilyapp.com, juste send. Cloudflare append le domaine automatiquement.

2. La valeur TXT SPF inclut les guillemets "v=spf1..." — Cloudflare les ajoute lui-même, donc colle sans guillemets.

8

Verify DNS Records

Une fois les 3 records posés côté Cloudflare (cf. section Cloudflare ci-dessous) :

[BOUTON] "Verify DNS Records"
Position haut-droite, fond noir
Action Resend interroge les DNS, badge passe à "Pending"

Statuts possibles (polling auto toutes ~30s)

  • not_started (gris) → tu n'as pas encore cliqué Verify
  • pending (jaune) → en cours, attendre 1-30 min (Cloudflare propage en ~1 min en général)
  • verified (vert ✓) → done, tu peux envoyer
  • partially_verified (orange) → un des 3 records OK, autres pending
  • failed (rouge) → après 72h sans détection

Polling status (automatique via prod-launch.sh)

Si verify timeout / fail

  1. Sur la page domain, clique chaque ligne pour voir "Detected: x" vs "Expected: y" — diff visible.
  2. Côté Cloudflare : vérifie le proxy status (gris obligatoire).
  3. Depuis PowerShell : nslookup -type=TXT resend._domainkey.fxmilyapp.com → doit renvoyer le p=....

☁️ Cloudflare — DNS + API Token

Tu possèdes déjà fxmilyapp.com chez Cloudflare. Trois tâches : créer un API token scopé, récupérer le Zone ID, et (en fallback) poser les records DNS Resend à la main.

⏱ ~5 min 📍 dash.cloudflare.com 🔑 Token format cfut_ (2026)
A

Créer le Custom API Token

  1. Ouvre https://dash.cloudflare.com/profile/api-tokens
  2. Bouton "Create Token" (haut-droite, bleu plein)
  3. Scrolle en bas → ligne "Create Custom Token" → bouton "Get started" (outline bleu, à droite de la ligne)

Formulaire

A. Token name : fxmily-deploy

B. Permissions — ajoute 2 lignes (bouton + Add more) :

  • Ligne 1 : Zone / Zone / Read
  • Ligne 2 : Zone / DNS / Edit

C. Account Resources : section probablement absente (pas nécessaire pour Zone-only permissions). Si présente, laisse "All accounts" par défaut.

D. Zone Resources :

  • 1er dropdown : Include (défaut, laisse)
  • 2e dropdown : Specific zone (PAS "All zones", sinon le token a accès à toutes tes zones)
  • 3e dropdown : fxmilyapp.com

E. Client IP Address Filtering : laisse vide (ton IP FAI peut changer)

F. TTL : End Date = aujourd'hui + 90 jours (rappel calendrier J-7)

Continuer

  1. "Continue to summary" → écran review
  2. "Create Token"
  3. Copie immédiatement (montré 1 fois). Format 2026 : cfut_xxxxxxxxxxxxx
  4. En bas : un curl de test → exécute pour valider, attendu "status":"active"
CLOUDFLARE_API_TOKEN=cfut_<copy-from-dashboard>
B

Récupérer Zone ID + Account ID

  1. Va sur https://dash.cloudflare.com
  2. Clique sur ta zone fxmilyapp.com dans la liste
  3. Tu arrives sur la page Overview (sélectionnée par défaut)

  5. Sur la droite, sidebar, sous "Account Details" / "Domain Registration", trouve la section "API"
  6. Deux lignes : Zone ID + Account ID (32 chars hex chacun)

C

Fallback — poser les DNS records manuellement

Tu fais ça uniquement si bootstrap-fxmily.sh a échoué. Sinon saute cette section, le script automatise tout via API.

Étape 0 — récupérer les valeurs Resend

  1. Onglet 1 : https://resend.com/domains → click sur fxmilyapp.com
  2. Tu vois le tableau 3 lignes (cf. Resend étape 7)
  3. Garde l'onglet ouvert

Étape 1 — DNS Cloudflare

  1. Onglet 2 : https://dash.cloudflare.com → fxmilyapp.com → DNS → Records
  2. Bouton "Add record" (haut-droite, bleu plein)

Étape 2 — A record pour app

  • Type : A
  • Name : app (juste app, Cloudflare append .fxmilyapp.com auto)
  • IPv4 address : 178.104.39.201 (ton Hetzner)
  • Proxy status : toggle GRIS = "DNS only" (V1 recommandé) ⚠️
  • TTL : Auto
  • Save
Proxy gris vs orange

V1 = DNS only (gris). Pourquoi : Caddy gère TLS sur le serveur. Activer le proxy orange = double TLS = risque erreur 525. V2 si stable, tu pourras passer en orange (DDoS shield).

Étape 3 — MX pour send

  • Type : MX
  • Name : send
  • Mail server : valeur exacte depuis Resend (ex. feedback-smtp.us-east-1.amazonses.com — ou eu-central-1 si Pro EU)
  • Priority : 10
  • Proxy : MX n'a pas de toggle (CF ne proxifie jamais MX)
  • Save

Étape 4 — TXT SPF pour send

  • Type : TXT
  • Name : send
  • Content : v=spf1 include:amazonses.com ~all (sans guillemets, CF les ajoute)
  • Save

Étape 5 — TXT DKIM pour resend._domainkey

UNE seule entrée TXT — pas 3 CNAMEs

Resend signe via sa propre clé, pas via les sous-clés SES. Si tu as posé 3 CNAMEs *.dkim.amazonses.com ailleurs, supprime-les.

  • Type : TXT
  • Name : resend._domainkey
  • Content : la longue clé publique copiée depuis Resend (commence par p=..., ~210 chars)
  • Save

Étape 6 — TXT DMARC pour _dmarc (optionnel mais recommandé)

  • Type : TXT
  • Name : _dmarc
  • Content : v=DMARC1; p=none; rua=mailto:eliot@fxmilyapp.com
  • p=none = mode observation V1 ; tu durciras en quarantine plus tard
  • Save

Étape 7 — Vérifier dans la liste

#TypeNameProxy
1Aapp.fxmilyapp.comDNS only (gris)
2MXsend.fxmilyapp.com—
3TXTsend.fxmilyapp.com—
4TXTresend._domainkey.fxmilyapp.com—
5TXT_dmarc.fxmilyapp.com—
Règle d'or

Sur les lignes mail, le proxy DOIT être DNS only. MX/TXT n'ont pas de toggle (donc auto-conforme). Si tu rajoutes un CNAME mail un jour, vérifie qu'il est gris. Un nuage orange sur DKIM = signature cassée = mails en spam.

Étape 8 — Retour Resend pour valider

Onglet Resend → page domain → bouton "Verify DNS Records". Propagation 1–10 min via Cloudflare en général. Si rouge après 30 min : recopie les valeurs (un caractère manquant suffit à casser).

📖 Glossaire débutant

Termes techniques que tu rencontres souvent dans Sentry / Resend / Cloudflare et qui peuvent être obscurs au départ.

DNS (Domain Name System)

Annuaire qui traduit fxmilyapp.com en adresse IP du serveur. Tu y poses des "records" (entrées). Cloudflare est ton DNS provider.

A record

Record DNS qui pointe un nom (ex. app.fxmilyapp.com) vers une IP IPv4 (178.104.39.201). C'est ce qui permet à un browser de trouver ton serveur.

MX record

Mail eXchanger. Indique quel serveur mail accepte les emails pour ton domaine. Resend en pose un sur send.fxmilyapp.com pour gérer les bounces (rebonds).

TXT record

Record DNS qui contient du texte arbitraire. Utilisé pour SPF, DKIM, DMARC, vérifications de propriété de domaine.

CNAME record

Record qui pointe un nom vers un autre nom (alias). Resend N'utilise PAS de CNAME pour DKIM (contrairement à SES direct).

SPF (Sender Policy Framework)

Standard qui dit "quels serveurs ont le droit d'envoyer des emails au nom de ce domaine". Sans SPF correct, Gmail/Outlook flag tes emails en spam.

DKIM (DomainKeys Identified Mail)

Standard qui signe cryptographiquement chaque email avec une clé privée. Le récepteur vérifie avec la clé publique posée en TXT DNS. Sans DKIM, livraison dégradée.

DMARC

Standard qui dit "que faire si SPF + DKIM échouent ?" (none = observe, quarantine = spam, reject = drop). Recommandé d'activer même en mode p=none.

DSN (Data Source Name)

URL Sentry qui identifie ton projet. Côté app, on l'utilise pour envoyer les erreurs à la bonne organisation/projet Sentry. Format : https://hash@oXXX.ingest.de.sentry.io/N.

API token

Mot de passe long pour authentifier les appels API. Scopé (limité à certaines actions). Si fuité, à révoquer immédiatement.

Auth Token (Sentry)

Token spécifique Sentry pour les actions CI/CD : upload sourcemaps, créer releases, etc. Format sntrys_eyJ....

Zone ID (Cloudflare)

Identifiant unique d'un domaine dans Cloudflare. Utilisé par les API pour cibler les records DNS. 32 chars hex.

Proxy status (Cloudflare)

Toggle DNS only (gris) vs Proxied (orange). Orange = trafic passe par les serveurs Cloudflare (DDoS shield, cache, WAF). Gris = trafic direct vers ton serveur. Pour records mail, TOUJOURS gris.

Region / Data residency

Où sont stockées physiquement tes données. Pour RGPD France, choisis EU/Frankfurt sur Sentry et Resend Pro. Sur Resend Free, pas de choix (US par défaut).

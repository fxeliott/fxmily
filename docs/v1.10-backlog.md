## TIER A — Hardening Security (2026 best practice)

### A1. Pin all GitHub Actions to commit SHA

**Why deferred** : ~16 actions tagged `@v3-7` instead of full SHA.
Supply-chain attacks on `tj-actions/changed-files` (Mar 2025, 23k repos
affected) + `aquasecurity/trivy-action` (76 of 77 tags moved to
malicious commits) + LiteLLM PyPI cascade demonstrate the threat.
Per [Wiz 2026 guide](https://www.wiz.io/blog/github-actions-security-guide),
only **3.9 % of repos pin 100 % of third-party Actions** — Fxmily is
in the unprotected 96.1 %.

**Tools** :

- [`pinact`](https://github.com/suzuki-shunsuke/pinact) CLI :
  `pinact run` rewrites every `uses: x@v4` line to `uses: x@SHA  # v4.x.y`.
  Companion [`pinact-action`](https://github.com/suzuki-shunsuke/pinact-action)
  enforces it in CI (`pinact run --check`).
- [Renovate `helpers:pinGitHubActionDigests`](https://docs.renovatebot.com/modules/manager/github-actions/)
  — opens a single PR converting all tags to SHAs + maintains updates
  ongoing. **Caveat** : Fxmily uses Dependabot today, adding Renovate
  creates competition. Stick with pinact + manual updates via
  Dependabot.

**Effort** : ~30 min (install pinact via aqua/go/brew, run, review diff,
commit, PR, add CI workflow with `pinact-action --check`).

**Suggested order** : do this **first** in V1.10, low-risk + automated.

### A2. Run `zizmor` static analysis on `.github/workflows/`

**Why deferred** : V1.9 hygiène scope. `zizmor` detects unpinned
actions, suspicious patterns, missing `permissions` blocks, dangerous
triggers (`pull_request_target` without guard). Should be added as a
CI step alongside `pinact-action`.

**Effort** : 10 min (add workflow that runs `zizmor .github/workflows/`).

contrast tokens cross-cutting risks breaking the `--t-3` tone hierarchy
on eyebrows, hints, and captions outside counter scope.

### C1. Char counter contrast `--t-3` → `--t-2` (WCAG 1.4.3 AA)

**Why deferred** : 50+ `text-[var(--t-3)]` occurrences across V1.8
wizards — refactor scope cross-cutting, risk regression on tone
hierarchy.

**Effort** : 1 h scoped fix (only counter elements, not all `--t-3`).

### C2. TradeTagsPicker `aria-describedby` refactor

**Current** : `aria-live="polite"` on the tooltip aside → SR verbose
on 8 hovers consecutifs (announces each focus change).

**Target** : APG tooltip pattern — permanent SR-only `<span
id="tag-desc-${slug}">` per tag + `aria-describedby={`tag-desc-${slug}`}`
on switch. SR reads description on focus, not on hover storm.

**Effort** : 1.5 h refactor + RTL test update.

### C3. EmptyState DS adoption on wizards

V1.8 wizards use `<div border-dashed>` ad-hoc empty states. J7 has
`<EmptyState>` DS component with illustration + heading + CTA. Adopt
across `/review`, `/reflect`, `/journal` empty cases.

**Effort** : 2 h.

### C4. Hero illustrations richesse

## TIER D — DS coherence (V1.9 audit deferred)

### D1. OKLCH literals → `@theme` tokens

**Current** : `oklch(0.62 0.19 254 / 0.14)` repeated 50+ times hardcoded
in V1.8 components. Per [Tailwind v4 docs](https://tailwindcss.com/blog/tailwindcss-v4),
`@theme` directive is the canonical pattern — single source of truth
in CSS variables, accessible via DevTools + JS + AI tools.

**Refactor** : create 5-7 alias tokens (`--v18-accent-bg-soft`,
`--v18-accent-text`, `--v18-border-accent`, etc.) in `.v18-theme`
overlay block, then `s/oklch(0.62 0.19 254 \/ 0.14)/var(--v18-accent-bg-soft)/g`.

**Effort** : 1.5 h + RTL no-regression run.

### D2. CTA réinventés vs `<Btn>` DS adoption

8 sites duplicate 50+ Tailwind classes when `<Btn kind="primary"
size="m|l">` exists. Pattern Phase P J10 welcome/admin/members already
demonstrates the migration. Apply to V1.8 wizards.

**Effort** : 1 h.

### D3. Spring values incohérents

`damping 30 mass 0.6` (step-progress) vs `damping 28 mass 0.7`
(wizards) vs `easeInOut` raw (heroes). Extract `V18_SPRING` +
`V18_EASE_DRAW` consts in `v18/motion-presets.ts`.

**Effort** : 30 min.

### D4. Magic spacing 2.5/1.5 hors 4-pt grid

`space-y-2.5` (10 px) / `gap-1.5` (6 px) sortent du DS-v2 doctrine
"4-pt strict". 9 sites à remplacer par `space-y-3` / `gap-2`.

**Effort** : 30 min.

### D5. Typography `clamp` override

## TIER G — V2.x Long-term considerations

### G1. Better Auth migration (Auth.js v5 successor)

**Context** : Auth.js v5 in security-patch mode since Sept 2025 — the
[Better Auth team took over maintenance](https://github.com/nextauthjs/next-auth/discussions/13252).
For new projects, the Auth.js docs themselves point to Better Auth.
For existing v5 projects (Fxmily), nothing breaks — no migration
deadline. **Defer to V2.x** when other priorities allow.

**Migration cost** : significant — DB schema changes (`provider` →
`providerId`, new `Verification` table replacing `VerificationToken`,
`password` field built-in for credentials), helper API (`signIn`
becomes `authClient.signIn.social`), middleware swap. Plan for
1-2 week dedicated session.

**Reference** : [Better Auth migration guide](https://better-auth.com/docs/guides/next-auth-migration-guide).

### G2. React 19 `use cache` directive + Cache Components PPR adoption

Next.js 16 stabilized Cache Components with Partial Pre-Rendering.
`use cache` directive enables granular caching of Server Component
fragments. Combined with `<Suspense>` for dynamic data, gives "static
shell + streaming" model. **Audit V2.x** : identify which dashboard
pages benefit (probably `/dashboard` + `/library` static catalog).

**Effort** : ~1 day audit + selective adoption.

### G3. Anthropic compaction beta `compact-2026-01-12`

For `weekly-batch-local.sh` (V1.7.2 HTTP migration), if context grows
beyond 50 k tokens (currently far below at 30 members × 7 days), the
compaction beta auto-summarizes older history. Add `betas:
['compact-2026-01-12']` + `context_management` param when relevant.

**Effort** : 30 min wire + cost monitoring.

### G4. PgBouncer transaction-mode (V2 scale)

Currently Hetzner Postgres direct (no pooler). Per Prisma 7 docs +
production checklist, PgBouncer transaction mode + `?pgbouncer=true`
URL param disables Prisma's prepared statement cache (incompatible
with PgBouncer < 1.21). At 30 members, direct connection is fine. At

> 100 members, add PgBouncer.

**Effort** : 1 day infra setup + Prisma config change + smoke tests.

### G5. iOS PWA Declarative Web Push validation real-device

V1.8 ships Apple Declarative Web Push (Safari 18.4+) per [Apple docs](https://developer.apple.com/documentation/usernotifications/sending-web-push-notifications-in-web-apps-and-browsers).
Need real-device validation : install PWA on iPhone iOS 18.4+ →
trigger push → verify lock screen + notification center display.

**Effort** : 30 min device test (Step 6 user-side checklist).
